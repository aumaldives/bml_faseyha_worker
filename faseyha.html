Force-update API

Force-fetch an account's history and push it to the configured callbacks even when no balance change was detected. The Force buttons above call this; you can also call it directly. It's firewall-gated (no app password) and uses the saved BML session, so the account id is resolved server-side from the number.

GET | POST  /api/forceupdate/{accountNumber}

Example

curl -X POST /api/forceupdate/7730000175583

Response (includes the fetched history)

{ "ok": true, "account": "7730000175583", "id": "…", "transactions": 12,
  "callbacks_enabled": 1, "posted_ok": true,
  "history": [ { "id": "FT…", "amount": 100, "narrative3": "…", … }, … ] }

Settings

Transaction-history callbacks — each enabled one is fired on a balance change. Up to two destinations.

Notifications (ALERTS) callbacks — each enabled one is POSTed the full response only when the alert set changes. Up to two destinations.

Firewall — allowed IPs
